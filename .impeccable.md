# Tetris — Design Context

## Users

A solo player loading the page to play a few rounds. They already know how Tetris works. The UI should stay out of the way.

## Aesthetic Direction

Two themes — `light` (default) and `dark`. Same shapes, same motion, same typography; only the surface palette flips.

- Display + body: **Hanken Grotesk**.
- Numerals + labels: **JetBrains Mono** for tabular alignment.
- Cells use the standard Tetris piece colours, lightly desaturated in dark mode for surface contrast.

## Design Principles

1. **Tokens, not magic numbers.** Every colour, font, radius, and animation duration lives in a CSS variable on `:root` (light) and is overridden in `[data-theme='dark']`. No hard-coded colours in components.
2. **Domain stays pure.** `PieceType` knows nothing about colour. Visual mapping lives in the UI layer.
